# Reference — `ggui.json` (the ggui server config, incl. theming)

> Local reference for the agent working in this template. `servers/ggui/ggui.json`
> is the operator config `ggui serve` reads. It shapes the **shell** of the app
> (model, theme, gadgets) — the UI itself is generated by the agent at runtime.

## Shape (the default shipped here)

```json
{
  "schema": "1",
  "protocol": "draft-2026-05-23",
  "app": {
    "slug": "sample-ggui-default",
    "name": "Sample ggui — default"
  },
  "generation": {
    "model": "anthropic:claude-haiku-4-5-20251001"
  },
  "theme": {
    "preset": "indigo",
    "mode": "dark"
  }
}
```

## Fields

### `generation.model` — what generates the UI
`"<provider>:<model>"` (e.g. `anthropic:claude-haiku-4-5-20251001`). This is the
model ggui uses to generate the React UI — separate from the model your **agent**
runs on (that's the agent's `MODEL` env). Bump to a stronger model for richer
generation. Both `provider:model` and `provider/model` forms are accepted.

### `theme` — how the app looks
A **pointer**, in one of two forms:
- **Preset:** `{ "preset": "indigo", "mode": "dark" }` (or the string shorthand
  `"theme": "indigo"`).
- **Custom:** `{ "file": "./theme.json" }` — your own DTCG theme document.

Changing it restyles **every** agent-generated surface with zero UI code
(two-layer theming: design tokens → CSS variables). **Full guide — the preset
list, light/dark, and authoring a custom brand `theme.json` (+ `ggui theme
validate`) — is in [`.reference/theming.md`](./theming.md).**

### `app` — identity + capabilities
- `slug`, `name` — your app's id + display name.
- `defaultDisplayMode` — optional; e.g. `"fullscreen"` for canvas-style apps.
- `gadgets` — client-side libraries the generator may use (maps, charts, …).
  Each: `{ package, version, requires?, connect?, exports: [...] }`. Author with
  `/gadget`; see the leaflet/mapbox samples for the shape.
- `publicEnv` — `{ "GGUI_PUBLIC_APP_<X>": "<value>" }`. Browser-exposed env a
  gadget needs (e.g. a Mapbox token). Required before any UI using that gadget
  can render.

## Deploy note
On a remote host, the ggui server must also know its own **public URL** so the
browser can load its runtime bundle + live channel. The deploy flow sets
`GGUI_PUBLIC_BASE_URL` for that (handled by `pnpm deploy:railway`). Locally it
defaults to `http://localhost:6781`, which is why local dev "just works."
